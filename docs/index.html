Claude Code plugin

A live ETA for your Claude Code runs.

The moment you submit a prompt, you get an instant range. As Claude works, a statusline countdown ticks down — and widens the second the run hits trouble: an error surfaces, a test fails, a debug spiral starts.

at submit
⏱ est. ~1m–6m
Runs entirely on your machine. No network calls.

Why

Agent runs are heavy-tailed.

Most of what decides how long a run takes is work that doesn't exist yet at the moment you hit enter — you can't know there's a failing test three tool calls out. So claude-eta never shows a fake-precise number. It shows a range up front, then sharpens that range as real evidence comes in: which phase Claude is in, how many tool calls it's made, and how many errors it's hit along the way.

Treat it as a sense of scale, not a stopwatch.

What you'll see

An estimate that reacts to the run, not just the clock.

At submit

An instant range based on your prompt's length, file references, and intent — plus the model and mode you're running.

While Claude works

A statusline countdown that shrinks as time passes, and jumps back up when the run hits trouble. 

⏱ ~3m10s left ███░░░░░ dbg +2⚠

dbg is the phase Claude's currently in (exp / edit / tst / dbg / oth). +2⚠ means two issues have shown up since submit — that's what just widened the ETA.

Install

Two commands, then reload.

  1. /plugin marketplace add https://github.com/ben564885/claude-eta

    Use the full https:// URL — the shorthand form clones over SSH and fails without keys already configured.

  2. /plugin install claude-eta@claude-eta

    Choose user scope when asked — it's already highlighted, so just press enter.

  3. /reload-plugins

    That's it for the instant estimate. Next prompt you submit shows ⏱ est. ~1m–6m.

Get the live countdown

One manual step — plugins can't install a statusline yet.

Add a statusLine key to ~/.claude/settings.json pointing at the installed statusline.mjs, then restart Claude Code. Without this step claude-eta still works fine — you just get the instant estimate, not the ticking countdown.

Calibration

It learns from your own runs.

Every finished run reports its predicted-vs-actual time back to a small local calibration file. claude-eta tracks a running correction factor per model-and-mode combination — opus:normal runs consistently longer than predicted, so future estimates for it scale up — blended with a global fallback so one noisy run doesn't overcorrect everything. 

claude-eta — 14 runs logged

  median error:                  38%
  median actual/predicted ratio: 1.12x

  learned calibration (global): 1.15x over 14 runs

  by model:mode bucket:
    opus:normal        1.34x  (n=6)
    sonnet:normal       0.92x  (n=8)

Check it any time with /eta-stats, or wipe the calibration and start over with /eta-reset.

How it works

Three hooks, one small state file per session.

UserPromptSubmit

Computes the instant estimate from heuristics and learned calibration, writes fresh session state. Pure local computation.

PostToolUse

Fires after every tool call, classifies it into a phase — explore, edit, test, debug, other — and revises the estimate as issues and phase time accumulate.

Stop

Closes out the run, appends one row to the local history file, and reports the predicted-vs-actual outcome back to the calibration file.

Zero runtime dependencies, zero network calls — plain Node.js reading and writing local JSON under ~/.claude/eta/.

Privacy

Only the shape of a run is recorded.

Model, mode, prompt length, file-reference count, phase timings, tool and issue counts, durations, and the predicted-vs-actual timing used for calibration. Prompt text and tool output are never stored, anywhere. Nothing leaves your machine.
